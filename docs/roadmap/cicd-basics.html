CI/CD Fundamentals

Automating the path from commit to production.

Continuous Everything

CI/CD is not just a tool (like Jenkins or GitHub Actions); it's a culture of frequent, small, and automated releases.

The Pipeline Stages

  • Build: Compiling code and resolving dependencies.
  • Test: Running automated test suites and static analysis.
  • Deploy: Pushing artifacts to environments (Dev → Stage → Prod).
  • Post-Deploy: Running smoke tests and verifying health.

Example: GitHub Actions Workflow

name: CI Pipeline
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Tests
        run: npm test
