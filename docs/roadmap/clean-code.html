Clean Code

Infrastructure as Code requires engineering rigor.

Why Clean Code in DevOps?

When "servers are just files," the quality of those files determines the stability of your production environment. Clean code reduces technical debt and makes architecture changes safer.

Key Principles

  • DRY (Don't Repeat Yourself): Use terraform modules or python utility functions instead of copy-pasting code.
  • Meaningful Naming: `db_subnet` is better than `subnet_101`. Always prioritize clarity over brevity.
  • Idempotency: Code should be safe to run multiple times without changing the final state unnecessarily.

Example: Resource Naming

# Bad: Generic naming
resource "aws_vpc" "vpc1" { ... }

# Good: Context-aware naming
resource "aws_vpc" "production_core_vpc" { ... }
